PREPROCESSING STEPS

4. UNIVERSAL TABLE PROCESSING

APPLY ONLY IF:

GENERAL TABLE-TO-TEXT LOGIC:

Step 1: Extract structure
- Identify headers (first row or inferred column labels)
- Identify rows

Step 2: For each row:
- Map column → value pairs
- Convert into natural language

Step 3: Sentence generation rules (GENERIC):

- Use pattern:
  "<Column> is <Value>"

- If multiple columns:
  Combine with commas and "and"

- If meaningful key exists (like Year, Name, Product):
  Use it as subject:
  Example:
  "In 2023, revenue was 100M."
  "Product A had sales of 200 and profit of 50."

- If no clear structure:
  Fallback:
  "Column1 is Value1, Column2 is Value2."

Step 4: Combine all row sentences into paragraph text

Step 5: Append to page text

IMPORTANT:
- Must work for ANY table structure
- Must not depend on fixed column names
- Must produce readable sentences

LARGE TABLES (cells ≥ 100):
- COMPLETELY IGNORE

5. MERGE TABLE TEXT

6. TEXT CLEANING

- Normalize punctuation
- Ensure readability

TAB 5: CHUNKING (FINAL RAG STAGE)

CHUNKING STRATEGY


1. PAGE-LEVEL SEGMENTATION
- Process each page independently
- Preserve page number

2. SENTENCE SPLITTING
- Split text into sentences

3. CONTEXT-BASED GROUPING (CORE LOGIC)
- Group sentences into meaningful chunks
- Start new chunk when:
  - topic changes
  - contrast words appear (however, but, etc.)
  - semantic shift occurs

4. CHUNK SIZE RULES
- 2–5 sentences per chunk
- Ideal: 300–400 tokens
- Max: 500 tokens

If too large:
→ split at sentence boundary

If too small:
→ merge with adjacent chunk

5. SLIDING WINDOW (MANDATORY)
- Overlap last sentence with next chunk

Example:
Chunk 1 → S1 S2 S3
Chunk 2 → S3 S4 S5

6. STRICT PAGE CONTROL
- Only use self.cleaned_pages_data
- Do NOT access other pages
